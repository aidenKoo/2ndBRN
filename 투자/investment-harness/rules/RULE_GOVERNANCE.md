# Rule Governance

## 목적
모든 투자 판단 룰은 기록되고, 버전 관리되어야 한다.

## 룰 변경 조건
다음 중 하나 이상 충족 시 변경 제안 가능.

1. 백테스트 실패 패턴 반복
2. 특정 시나리오 손실 과다
3. 리스크 한도 초과 반복
4. 실제 성과와 기대 성과의 괴리 발생
5. 메타 분석 결과 기존 룰의 비효율 확인
6. 지식 아카이브에서 검증 가능한 룰 후보 도출

## 금지 사항
- 감정적 시장 판단에 따른 즉시 룰 변경 금지
- 단일 사례만으로 핵심 룰 변경 금지
- 백테스트 없는 룰 승격 금지
- 변경 사유 없는 active 룰 수정 금지

## 승인 단계
draft → proposed → tested → approved → active → archived

## 룰 변경 후 필수 작업
1. 기존 active 룰 백업
2. versions 폴더에 스냅샷 생성
3. CHANGELOG 작성
4. 관련 롤에게 피드백 공유
5. 다음 리뷰에서 성과 확인
