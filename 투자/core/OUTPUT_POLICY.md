# Output Policy

## 목적
모델별 차이를 줄이고 재사용 가능한 출력 형식을 유지한다.

## 표준 출력 형식
1. 현재 판단 (한 줄)
2. 상승 논리 (2~4개)
3. 하락 논리 (2~4개)
4. 행동 제안 (분할/관망/축소 등)
5. 무효화 조건
6. 다음 체크포인트
7. 이론 커버리지 점수 (High/Medium/Low)

## 품질 체크리스트
- 반대 논리가 포함되어 있는가?
- 리스크와 무효화 조건이 숫자/조건 형태로 명확한가?
- 데이터 최신성 한계가 명시되었는가?
- 최소 3개 이론 축(팩터/밸류/리스크 등)이 반영되었는가?
